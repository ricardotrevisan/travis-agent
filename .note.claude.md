## Changes Applied 2026-03-12

### app.py вЂ” Redis dedupe in app layer
- `_mark_message_processed()` and `_redis` now live in `app.py`, removing the `engine` dependency.
- Redis connection initialized at startup with graceful fallback (`None` if unavailable).
- Dedup key = `idempotency_key` or `task:{task_id}`, TTL via `WEBHOOK_DEDUPE_TTL_SECONDS` (default 300s).

### runtime/orchestrator.py вЂ” SkillRegistry lazy singleton
- `SkillRegistry` no longer instantiated on every webhook call.
- Module-level singleton via `_get_registry()` / `_reset_registry()`.
- `/new` reset calls `_reset_registry()` + `invalidate_mcp_tools_cache()`.

### runtime/executor.py вЂ” n8n bypass for LLM post-processor
- Guard: if any plan step is `n8n_schedule_alert`, final LLM interpreter is skipped.

### runtime/planner.py вЂ” explicit run_at rule
- Prompt rule added: planner must include `args.run_at` with ISO 8601 datetime for n8n create.

### skills/n8n_schedule_alert.py вЂ” datetime resolution + fixes
- `_ISO_8601_NO_TZ_RE` added: handles datetimes without TZ, localized via `SCHEDULE_TIMEZONE`.
- `_DMY_TIME_RE` now requires time component (bare dates no longer match).
- `_format_success_text`: warns explicitly when n8n responds without `idTask`.
- `create` datetime resolution order:
  1. `args.run_at`
  2. `args.datetime`
  3. `args.time`
  4. generic scan of all string arg values for ISO 8601
  5. fallback: regex extraction from `ctx.user_text`

## Datetime Formats Recognized by n8n_schedule_alert

| Format | Example | Notes |
|---|---|---|
| ISO 8601 with TZ | `2026-03-12T19:28:00-03:00` | Converted to UTC |
| ISO 8601 UTC | `2026-03-12T19:28:00Z` | Used as-is |
| ISO 8601 no TZ | `2026-03-12T19:28:00` | Localized via SCHEDULE_TIMEZONE |
| ISO no TZ space | `2026-03-12 19:28` | Localized via SCHEDULE_TIMEZONE |
| Day/month + time | `12/03 19:28`, `12-03 Г s 19:28` | Current year assumed |
| Day/month/year + time | `12/03/2026 19:28` | Full date |
| Relative hoje + time | `hoje Г s 15:30` | Today |
| Relative amanhГЈ + time | `amanhГЈ 10:00` | Tomorrow |

**Does NOT work (triggers clarification):** time-only (`Г s 19h`), weekday names (`sexta`), written months (`15 de marГ§o`), relative durations (`daqui a 30 min`).
